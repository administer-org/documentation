# Documentation Issues and Contributing

## Issues

If you have spotted an issue with any of the documentation here (typo, incorrect example, incomplete output, old code, etc), then please file an issue on the repository: https://to.admsoftware.org/git/documentation

## Contributions

If you would like to contribute new docs (tutorials, examples, app docs, etc), then feel free to open a pull request. As with all of our other code, we accept pull requests at our sole discresion, but they will generally be accepted as long as they are useful and can be seen as useful, simple, and easy to understand for everybody. We use [Vitepress](https://vitepress.dev) for our docs so their config rules apply.
